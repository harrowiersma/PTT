01 Voice Communication (PTT)

Voice Push-to-Talk

Two-way voice communication using the Mumble protocol. Press the hardware PTT button on the device to transmit. Release to listen. Audio is encoded with Opus at 16–24 kbps for clear voice quality with low latency (<200 ms).

Server: Murmur (Mumble server) handles voice routing, encoding, and channel management.

Client: openPTT app on Android devices (Hytera P50, or any Android device with PTT support). Forked from Mumla, includes built-in GPS reporting and hardware-key handling for rugged radios.

Voice PTT triple-tap gestures

Three quick PTT presses within 1.2 seconds trigger a special action. Currently:

• Triple-tap PTT → toggle lone-worker shift start/stop (see Lone Worker Safety).

Normal single-press PTT behaviour is unaffected — you can still talk normally.

02 Channels

Voice Channel management

Create channels for different teams or purposes (Warehouse, Security, Office). Users join a channel to hear and talk to other members. Managed from Directory → Channels (Add form + per-row delete). The Channels stat card on the Live Ops Overview is also clickable as a shortcut.

Special channels:

• Emergency — Auto-created for SOS events. All users are moved here during emergencies.

• Weather — Double-PTT to hear a spoken weather report for your GPS location. The PTTWeather bot sits permanently in this channel on a dedicated Mumble connection so the main admin bot stays in Root for SOS acknowledgements.

• Phone / Call-1…Call-N — SIP gateway sub-channels (see SIP Phone Gateway).

03 GPS Tracking

Ops Live device tracking

GPS is reported by the openPTT app itself — no separate Traccar Client app needed. Worker positions appear on the embedded map under Live Ops → Dispatch (the standalone Map tab folded into Dispatch). The Live Ops Overview table also lists per-device GPS coordinates, battery level, and last-seen time.

Reporting cadence: every 30 seconds or 50 m of movement, whichever comes first. The device's Mumble username is used as the Traccar uniqueId, so Traccar devices and PTT users share identity automatically.

App setting: Traccar server URL — http://voice.harro.ch:5055 (plain HTTP because the OsmAnd protocol doesn't speak TLS; cleartext is whitelisted for that host only).

04 Emergency SOS

Safety Emergency alert system

When an SOS is triggered:

  1. All connected users are automatically moved to the Emergency channel
  2. Red alert banner appears on the admin dashboard with GPS location
  3. Webhook / email notification sent to configured admin contacts
  4. Everyone can hear the person in distress, regardless of which channel they were in

Acknowledge: An admin can acknowledge from the dashboard or type "OK" in the Emergency channel chat. All users are moved back to their original channels.

Trigger methods: API call (POST /api/sos) and Traccar event forwarding to POST /api/sos/traccar/event (man-down / panic button events generated on the device route through Traccar and into the admin service automatically).

05 Weather Channel (ATIS)

Ops Spoken weather reports

Aviation-style weather reporting. Join the Weather channel and press PTT twice quickly (within 10 seconds). The system:

  1. Looks up your GPS position from Traccar
  2. Fetches current weather and a 6-hour hourly forecast from Open-Meteo (free, no API key)
  3. Generates a spoken report using Piper TTS (en_US-lessac-medium, baked into the admin image)
  4. Plays the audio in the Weather channel

Report includes: temperature, wind speed and direction, cloud cover, current conditions, plus a 6-hour outlook mentioning expected rain and wind gusts when above threshold (≥ 0.5 mm total / ≥ 50 % probability / ≥ 40 km/h gust).

Rate limit: one request per user per 60 s.

Also available via API: POST /api/weather with {"location": "Paris, France"}

06 Dispatch

Ops Single-page dispatch workbench

The Live Ops → Dispatch page is the operator's whole workbench: quick-pick chips, search, embedded Leaflet map with live worker markers, ranked nearest-worker list, and recent-dispatch log — top to bottom on one screen.

Pick a target three ways, all routed through the same code path:

  • Click a saved-location chip at the top.
  • Type a place name or lat, lng in the search box and hit Find Nearest (place names are geocoded via Open-Meteo).
  • Click anywhere on the embedded map.

An amber target pin drops on the map and the Nearest Workers table populates with up to max_workers closest results, filtered by the optional search radius (both configurable on Dispatch Setup — see below). Distance is haversine straight-line in metres.

Ops Sending a dispatch

Click Dispatch on a worker row to open the send modal. Pick a canned message from the dropdown (auto-fills the textarea, still editable) or type free-text. Press Send.

Dispatch uses Piper TTS whispered directly to the target user's Mumble session — only they hear it, regardless of which channel they're in. The message is prefixed with throwaway words ("Attention. Dispatch for {user}.") so the receiver's Opus decoder has time to wake up before the real content arrives.

Fallback: if TTS or the whisper fails, dispatch falls back to a text message in the target user's current channel. The API response includes a delivery field (tts_whisper / text / none).

Recent Dispatches log at the bottom of the page shows the last 50 dispatches (newest first) with timestamp, target, message, and coords. Updates live after each send.

Admin Dispatch Setup

All dispatch configuration lives at System → Dispatch Setup on a separate page so the operational view stays uncluttered:

  • Map Home — default centre (lat / lng) and zoom level the embedded map opens at. Defaults to Lisbon (38.72, -9.14, zoom 11).
  • Search Behaviour — max workers (1–50, default 10) caps the nearest-list length; search radius in metres (0 = unlimited) filters out workers further than that from the target.
  • Saved Locations — add / delete the quick-pick chips. Coords accepted as lat, lng or as a place name (geocoded on save).
  • Canned Messages — pre-written dispatch text the operator picks from the modal dropdown. Sort order controls display order.

Both the operational page and Setup page disappear from the dashboard when the dispatch feature is disabled in Features.

07 Lone Worker Safety

Safety Shift-based monitoring

Lone-worker check-ins are scoped to explicit shifts. When a user starts a shift they're monitored; when the shift ends (manually or by auto-expiry), the checker ignores them. No 24/7 pings.

Start / stop a shift from the device: triple-tap the PTT button (3 presses within 1.2 s). The openPTT app fires POST /api/loneworker/shift/start or /stop and confirms via Android TTS:

• First triple-tap: "Shift started. Ends at HH:MM."

• Second triple-tap: "Ending shift. Long press again to cancel." — 3-second grace window; triple-tap again to cancel, or let it elapse to commit the end.

Shift duration: per-user field on the user edit form (1–24 h). Blank uses the global default from the Lone Worker config.

While a shift is active:

• The checker runs every 60 s, pinging users with missed check-ins via a TTS voice reminder: "Automated check-in for {name}, please respond."

• Shifts past their planned_end_at are auto-closed with end_reason=auto_expired

• Status shown on the Users tab: OK / Due soon / OVERDUE

Why triple-tap PTT, not F3 long-press: the P50's ROM owns F3 long-press and can't be intercepted by apps. The device-settings "Lone Worker" option on F3 is the ROM's own implementation and is disabled here; our shift system runs independently on the admin server.

08 SIP Phone Gateway

Ops Inbound calls → Mumble

A DIDWW SIP trunk terminates inbound calls into the Asterisk-backed sip-bridge container. Each call lands in a dedicated Phone/Call-N sub-channel (up to PHONE_MAX_CALLS concurrent, default 3). Beyond that, callers get 486 Busy.

Flow: INVITE → Answer → Piper greeting → AudioSocket bridge to a per-call PTTPhone-N Mumble bot. Ringback plays to the caller while no radio user has joined the sub-channel.

Who can answer: users with can_answer_calls = true are eligible. Non-eligible users who walk into Phone or any Call-N sub-channel get bounced back with a whispered "Phone channel requires call-answer permission."

Notification: every eligible user gets a whispered 880 Hz + 1175 Hz ding (re-pinged every 3 s) and a structured INCOMING_CALL|caller|Call-N text-whisper that the openPTT app intercepts to raise a full-screen incoming-call overlay with Answer / Decline buttons. Tapping Answer moves the user straight into Phone/Call-N; the Phone parent channel is intentionally hidden from the device's channel carousel since manually parking there has no effect.

Radio-side call control (while the user is in a Phone/Call-N sub-channel):

• Green button (KEYCODE_CALL) — toggle caller-mute. Radios still hear each other; caller hears silence. Press again to restore bidirectional audio. TTS confirms each toggle.

• MENU key (KEYCODE_MENU) — radio-initiated hangup. TTS confirms "call ended."

• The channel knob (F5/F6) is locked during an active call so a stray rotation can't move the user out of the call mid-conversation.

Editable greeting: the Piper TTS greeting is stored per-trunk in sip_trunks.greeting_text. Edit on System → SIP Gateway; a Save & Regenerate button re-renders the WAV and hot-pushes it to the sip-bridge container.

09 User Management

Admin Device user accounts

Create users for each device from the Users tab. Each user gets a username, password, and optional display name. The admin server auto-creates a matching Traccar device on user creation with uniqueId=<username> so GPS and voice identity stay unified.

QR code: Click QR to generate a QR code for quick manual device enrollment. Scan with the device camera to auto-configure the openPTT app.

Roles and flags:

  • Admin — Can acknowledge SOS from the radio (type "OK" in Emergency channel)
  • Lone Worker — Subject to shift-based check-in monitoring. When checked, also set shift_duration_hours (1–24) or leave blank for the global default.
  • Can answer SIP calls — ACL for joining Phone / Call-N sub-channels.
  • User — Standard radio user with no extra privileges

Murmur registration reset: if a user's cert has gone stale, "Reset Murmur registration" on the user edit form clears their registered row and restarts Murmur so they get the cert prompt again on next connect (~3 s disruption).

11 Device Provisioning

Important — where to run

The provisioning link runs on your laptop, not on the P50.

Plug the P50 in over USB with USB debugging on. The script drives adb from your Mac or PC to install the APK and seed credentials on the handset. The P50 has no keyboard and no way to run shell scripts — it's the target, not the executor.

Admin One-click provisioning link

From the user edit form → Generate provisioning link. The dashboard shows a short URL plus a QR code. Send the URL to the tech doing the install.

What the tech does on their laptop:

  1. Plug the P50 in via USB. Enable Developer Options → USB debugging. Accept the RSA fingerprint prompt on the handset.
  2. Open the short link in a browser; it downloads openptt-provision.sh (macOS / Linux) or openptt-provision.ps1 (Windows).
  3. Run the script: 
    # macOS / Linux
curl -fsSL https://ptt.harro.ch/p/<slug> -o setup.sh
bash setup.sh
    # Windows (PowerShell)
Invoke-WebRequest https://ptt.harro.ch/p/<slug> -OutFile setup.ps1
.\setup.ps1

What the script does, unattended:

  • Verifies exactly one P50 is connected over adb
  • Downloads and installs the signed openPTT APK
  • Disables any Hytera stock PTT client (hiTalk / HM Client) when present. Uses a precise package allowlist — leaves the PTT keycode-142 input method (com.poc.inputmethod.poc), Hytera Market, and Hytera Factory Mode alone. Reversible via pm enable.
  • Grants RECORD_AUDIO, ACCESS_FINE_LOCATION, ACCESS_COARSE_LOCATION runtime perms
  • Adds openPTT to the Doze whitelist and grants RUN_IN_BACKGROUND / RUN_ANY_IN_BACKGROUND, so the MumlaService survives long periods of screen-off without the ROM killing it
  • Seeds SharedPreferences (admin URL, Traccar URL, auto-connect, PTT input method)
  • Seeds the Humla mumble.db with the user's Mumble credentials
  • Launches MumlaActivity so the service registers and auto-connects

Note on our P50 fleet: the devices we ship do not come with a hiTalk client pre-installed. The Hytera packages on the stock image (com.hytera.hyterafactorymode, com.hytera.market, com.poc.inputmethod.poc) are intentionally left enabled — the last one is what turns the side PTT button into KEYCODE 142 that openPTT listens for. The hiTalk disable step is there for devices sourced elsewhere.

Prerequisites on the laptop:

  • adb in PATH (brew install --cask android-platform-tools on macOS; Android platform-tools on Windows)
  • sqlite3 in PATH (ships on macOS / most Linux; winget install SQLite.SQLite on Windows)

Token lifetime: 24 hours, single-use. The token is marked used when the script finishes successfully. Generate a new one for each device.

Idempotent: if a step errors, re-run the script — every step is safe to repeat. If the error persists, capture it and ping an admin.

12 Bulk Enrollment

Admin CSV upload

From the Admin tab, upload a CSV file to create multiple users at once. Download the template, fill in usernames and passwords, upload. Returns a ZIP file with QR codes for each device.

For unattended provisioning of new devices, prefer the per-device provisioning link flow — it seeds credentials and launches the app without the tech having to type anything into the P50.

13 Audit Log

Admin Action history

System → Audit Log shows every admin action: user creation / deletion, admin management, SOS events. Filterable by action type. Includes timestamp, admin username, and IP address.

System → Call Log is the SIP equivalent — every inbound call's start, slot assignment, answer, and end with caller-id and answering operator.

14 Feature toggles

Admin Enable / disable modules deployment-wide

System → Features exposes one switch per module: Lone Worker, SIP, Dispatch, Weather, SOS. Disabling a module:

  • Hides the module's tabs and inline UI from the dashboard immediately (no reload).
  • Returns 503 "Feature 'X' is disabled by the administrator" from any of that module's API routes.
  • Skips that module's background tasks at next admin restart (Weather bot, lone-worker checker, SIP poller).
  • Hides matching UI on the P50 app (capability cache refreshes every 10 minutes; force-refresh by restarting the app).

Database state is preserved — re-enabling restores full behaviour. SOS shows a confirmation dialog before disabling because turning it off removes emergency response from the system.

The capability state is exposed publicly at GET /api/status/capabilities so the P50 app and the dashboard can both read it without authentication.

15 Monitoring

Admin Prometheus metrics

GET /metrics returns Prometheus-compatible metrics: uptime, Murmur connection status, users online, total users, channels, SOS events.

16 Manual Device Setup

When the provisioning link isn't available

For most devices, use the one-click provisioning link. The manual path below is only for first-of-its-kind testing or recovery.

AppPurposeServerPort
openPTTVoice PTT + GPS + shift controlvoice.harro.ch443 (voice) / 5055 (GPS)

Sideload via adb:

adb install -r openptt-foss-release.apk

First-time settings in the app:

  • Favourites → add server voice.harro.ch:443 with the user's Mumble credentials
  • Settings → GPS tracking → Traccar server URL: http://voice.harro.ch:5055
  • Settings → Admin API → Admin server URL: https://ptt.harro.ch (required for triple-tap shift control)
  • Grant runtime permissions: ACCESS_FINE_LOCATION (GPS) and RECORD_AUDIO (voice). Example: adb shell pm grant ch.harro.openptt android.permission.ACCESS_FINE_LOCATION

Device settings: disable battery optimisation for openPTT. Enable background running. The app auto-connects on boot via its BootReceiver.

Hardware-button behaviour on the Hytera P50:

  • Side PTT button (keycode 142 via vendor broadcast): push to talk
  • Triple-tap PTT within 1.2 s: toggle lone-worker shift
  • Channel rotary knob (F5 / F6): previous / next channel
  • Green button (KEYCODE_CALL): mute toggle while in a Phone/Call-N sub-channel (SIP)
  • MENU key (KEYCODE_MENU): hangup while in a Phone/Call-N sub-channel (SIP)
  • Red / F3 / F4 keys: owned by the ROM, not intercepted
