Voice Communication (PTT)

Voice Push-to-Talk

Two-way voice communication using the Mumble protocol. Press the hardware PTT button on the device to transmit. Release to listen. Audio is encoded with Opus at 16-24 kbps for clear voice quality with low latency (<200ms).

Server: Murmur (Mumble server) handles voice routing, encoding, and channel management.

Client: openPTT TRX app on Android devices (e.g., Hytera P50, or any Android device with PTT support). Forked from Mumla, includes built-in GPS reporting and hardware-key handling for rugged radios.

Voice PTT Triple-Tap Gestures

Three quick PTT presses within 1.2 seconds trigger a special action. Currently:

• Triple-tap PTT → toggle lone-worker shift start/stop (see Lone Worker Safety).

Normal single-press PTT behaviour is unaffected — you can still talk normally.

Channels

Voice Channel Management

Create channels for different teams or purposes (e.g., Warehouse, Security, Office). Users join a channel to hear and talk to other members. Managed from the Channels tab.

Special channels:

• Emergency — Auto-created for SOS events. All users are moved here during emergencies.

• Weather — Double-PTT to hear a spoken weather report for your GPS location. The PTTWeather bot sits permanently in this channel on a dedicated Mumble connection so the main admin bot stays in Root for SOS acknowledgements.

GPS Tracking & Map

Operations Live Device Tracking

GPS is reported by the openPTT TRX app itself — no separate Traccar Client app needed. The Map tab shows all device locations on an interactive map (OpenStreetMap).

Reporting cadence: every 30 seconds or 50 m of movement, whichever comes first. The device's Mumble username is used as the Traccar uniqueId, so Traccar devices and PTT users share identity automatically (commit 2308f17).

Overview tab shows GPS coordinates, battery level, and last seen time per device.

App setting: Settings → GPS tracking → Traccar server URL: http://voice.harro.ch:5055 (plain HTTP because the OsmAnd protocol doesn't speak TLS; cleartext is whitelisted for that host only).

Emergency SOS

Safety Emergency Alert System

When an SOS is triggered:

1. All connected users are automatically moved to the Emergency channel

2. Red alert banner appears on the admin dashboard with GPS location

3. Webhook/email notification sent to configured admin contacts

4. Everyone can hear the person in distress, regardless of which channel they were in

Acknowledge: An admin can acknowledge from the dashboard or type "OK" in the Emergency channel chat. All users are moved back to their original channels.

Trigger methods: API call (POST /api/sos) and Traccar event forwarding to POST /api/sos/traccar/event (man-down / panic button events generated on the device route through Traccar and into the admin service automatically).

Weather Channel (ATIS)

Operations Spoken Weather Reports

Aviation-style weather reporting. Join the Weather channel and press PTT twice quickly (within 10 seconds). The system:

1. Looks up your GPS position from Traccar (resolves via User.traccar_device_id link, falls back to device name)

2. Fetches current weather and a 6-hour hourly forecast from Open-Meteo (free, no API key)

3. Generates a spoken report using Piper TTS with the en_US-lessac-medium voice (baked into the admin image at build time)

4. Plays the audio in the Weather channel

Report includes: temperature, wind speed and direction, cloud cover, current conditions, plus a 6-hour outlook mentioning expected rain and wind gusts when above threshold (≥0.5mm total rain / ≥50% probability / ≥40 km/h gust).

Rate limit: One request per user per 60 seconds.

Also available via API: POST /api/weather with {"location": "Paris, France"}

Dispatch

Operations Nearest Worker Dispatch

From the Dispatch tab, enter a location name or coordinates. The system geocodes the location and finds the nearest workers by GPS distance.

Click Dispatch to send the message. Dispatch uses Piper TTS whispered directly to the target user's Mumble session — only they hear it, regardless of which channel they're in. The message is prefixed with a short tone and throwaway words ("Attention. Dispatch for {user}.") so the receiver's Opus decoder has time to wake up before the real content arrives.

Fallback: if TTS or the whisper fails, dispatch falls back to a text message in the target user's current channel. The API response includes a delivery field (tts_whisper / text / none) so the dashboard can show which path fired.

Also on Map tab: Click anywhere on the map to dispatch to that location.

Operations Saved Locations

Pre-configure frequently used dispatch locations for one-click access. Add locations by name (auto-geocoded) or coordinates from the Dispatch tab.

Saved locations appear as quick-pick buttons at the top of the Dispatch tab. Click a button to instantly search for the nearest workers to that location.

Lone Worker Safety

Safety Shift-Based Monitoring

Lone-worker check-ins are scoped to explicit shifts. When a user starts a shift, they're monitored; when the shift ends (manually or by auto-expiry), the checker ignores them. No 24/7 pings.

Start / stop a shift from the device: triple-tap the PTT button (3 presses within 1.2 seconds). The openPTT TRX app fires POST /api/loneworker/shift/start or /stop and confirms via Android TTS:

• First triple-tap: "Shift started. Ends at HH:MM."

• Second triple-tap: "Ending shift. Long press again to cancel." — 3-second grace window; triple-tap again to cancel, or let it elapse to commit the end.

Shift duration: per-user field on the user edit form (1–24 hours). Blank uses the global default from the Lone Worker config (default_shift_hours).

While a shift is active:

• The checker runs every 60 seconds, pinging users with LoneWorkerShift.ended_at IS NULL and missed check-ins via a TTS voice reminder: "Automated check-in for {name}, please respond"

• Shifts past their planned_end_at are auto-closed with end_reason=auto_expired

• Status shown on the Users tab: OK / Due soon / OVERDUE

Note: The P50's ROM owns the F3 long-press and can't be intercepted by apps — that's why the gesture is triple-tap PTT instead. The device-settings "Lone Worker" option on F3 is the ROM's own implementation and is disabled here; our shift system runs independently on the admin server.

Admin setup:

• Admin tab → Lone Worker Safety: set check-in interval, default shift hours, warning threshold, and auto-SOS behaviour.

• Users tab → edit a user: toggle "Lone Worker" + set shift_duration_hours.

API: POST /api/loneworker/shift/start, POST /api/loneworker/shift/stop, GET /api/loneworker/shift/active.

User Management

Admin Device User Accounts

Create users for each device from the Users tab. Each user gets a username, password, and optional display name. The admin server auto-creates a matching Traccar device on user creation with uniqueId=<username> so GPS and voice identity stay unified.

QR Code: Click QR to generate a QR code for quick device enrollment. Scan with the device camera to auto-configure the openPTT TRX app.

Roles and flags:

• Admin — Can acknowledge SOS from the radio (type "OK" in Emergency channel)

• Lone Worker — Subject to shift-based check-in monitoring. When checked, also set shift_duration_hours (1–24) or leave blank for the global default.

• Can answer SIP calls — ACL for joining Phone sub-channels once the SIP gateway is deployed. Off by default.

• User — Standard radio user with no extra privileges

Linked Traccar device: dropdown lets you link a user to a specific Traccar device ID. Auto-populated on user creation but can be re-linked if the device is re-provisioned.

Admin Accounts

Admin Dashboard Administrators

Manage admin dashboard accounts from the Admin tab. Passwords are hashed with bcrypt.

Roles: Admin (full access) or Viewer (read-only dashboard).

Bulk Enrollment

Admin CSV Upload

From the Admin tab, upload a CSV file to create multiple users at once. Download the template, fill in usernames and passwords, upload. Returns a ZIP file with QR codes for each device.

Audit Log

Admin Action History

The Audit Log tab shows every admin action: user creation/deletion, admin management, SOS events. Filterable by action type. Includes timestamp, admin username, and IP address.

Monitoring

Admin Prometheus Metrics

GET /metrics returns Prometheus-compatible metrics: uptime, Murmur connection status, users online, total users, channels, SOS events.

Health check: GET /api/status/health returns {"status": "healthy", "murmur": "connected"}

Device Setup

Device Configuration

Only one app needed:

AppPurposeServerPort
openPTT TRXVoice PTT + text + GPS + shift controlvoice.harro.ch443 (voice) / 5055 (GPS)

Sideload via adb: adb install -r openptt-foss-debug.apk

First-time settings in the app:

• Favourites → add server voice.harro.ch:443 with the user's Mumble credentials

• Settings → GPS tracking → Traccar server URL: http://voice.harro.ch:5055

• Settings → Admin API → Admin server URL: https://ptt.harro.ch (required for triple-tap shift control)

• Grant runtime permissions: ACCESS_FINE_LOCATION (GPS) and RECORD_AUDIO (voice). adb shell pm grant ch.harro.openptt android.permission.ACCESS_FINE_LOCATION skips the dialog.

Device settings: Disable battery optimization for openPTT TRX. Enable background running. The app auto-connects on boot via its BootReceiver.

Hardware button behaviour on Hytera P50:

• Side PTT button (keycode 142 via vendor broadcast): push to talk

• Triple-tap PTT within 1.2s: toggle lone-worker shift

• Channel rotary knob (F5/F6): previous/next channel

• Green/red/F3/F4 keys: owned by the ROM, not intercepted by the app
